3. Technical Architecture: The MERN Stack Strategy
Building a unicorn requires a tech stack that is not just modern but resilient, scalable, and SEO-friendly. The choice of MERN (MongoDB, Express.js, React.js, Node.js) with Next.js is optimal, but the implementation details determine success.

3.1 Frontend: Next.js & The Rendering Dilemma (SSR vs. CSR)
For an e-commerce platform, SEO is lifeblood. Pure React (Client-Side Rendering) fails at SEO because search engine crawlers struggle to index dynamic JavaScript content.   

Server-Side Rendering (SSR): We will use Next.js SSR for Product Detail Pages (PDP) and Listing Pages (PLP). This ensures that when a user searches for "Authentic Mysore Pak," Zaykaa’s page is fully rendered and indexed by Google, driving organic traffic.   

Client-Side Rendering (CSR): The User Dashboard, Cart, and Checkout flows will utilize CSR. These pages do not need SEO but require high interactivity and state persistence.

Static Site Generation (SSG): For "Static Content" like Blog posts ("History of the Samosa") and Artisan Stories, SSG will provide lightning-fast load times, improving Core Web Vitals.   

3.2 Backend: Microservices Architecture on Node.js
Monolithic architectures become nightmares as startups scale. We will adopt a modular structure within the Express.js framework.

Service Separation:

Auth Service: Handles OTP (Twilio/Firebase), OAuth (Google), and RBAC (Role-Based Access Control).

Order Service: Manages cart, inventory locking, and order state machine.

Catalog Service: Manages products, categories, and vector search.

Logistics Service: Integrates with Shiprocket API for AWB generation and tracking.   

API Security: Implement Rate Limiting (using Redis) to prevent DDoS attacks. Use Helmet.js for header security and Joi for input validation to prevent SQL/NoSQL injection.   

3.3 Database: MongoDB Schema Design for Multi-Tenancy
Supporting multiple vendors (home chefs) requires a sophisticated data model. We will use a Reference Data Model to ensure data consistency and scalability.   

Key Collections:

Users Collection: Stores diverse profiles (Customer, Vendor, Corporate, Admin) using a discriminator pattern or a role enum.

Tenants (Vendors) Collection: Stores artisan details, FSSAI licenses, and payout info.

Products Collection: Uses the "Attribute Pattern" to handle diverse snack attributes (e.g., specific allergens, shelf life, ingredients) without a rigid schema.   

Orders Collection: Must support "Split Orders" where a single customer cart contains items from multiple vendors. The system needs to split these into sub-orders for logistics fulfillment while presenting a unified view to the customer.

3.4 Integration Ecosystem
Logistics: Shiprocket API is the gold standard for D2C logistics in India. The integration will automate courier selection based on "Serviceable Pincodes" and "Lowest Cost/Fastest Delivery" logic.   

Payments: Razorpay or PhonePe integration to support UPI, which accounts for >90% of food delivery transactions.   

Notifications: WhatsApp Business API (Interakt or Wati) for real-time order updates. "Your Zaykaa Box has left the kitchen!" messages have higher open rates than email.
